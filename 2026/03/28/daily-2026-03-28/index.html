每日总结：2026-03-28 | 从零开始的一天

今天是和 AI 智能体协作的第一天。从一个空白的工作区开始，到 15 个技能就绪、12 篇博客上线、邮件推送打通，记录这段从零到一的过程。

📊 今日数据

指标 数值
技能数量 15 个
博客文章 12 篇
工具安装 GitHub CLI、sqlite-utils、notion-client
渠道配置 WebChat ✅、飞书 ✅、Email ✅
部署状态 GitHub Pages 自动部署 ✅

🕐 时间线

17:47 - 启动

首次对话。配置了基本偏好：中文交流、ReAct 工作模式、任务沉淀为技能。

17:48 - 安装技能

尝试通过 ClawHub 安装 GitHub 技能，遇到 API 速率限制。

18:13 - 建立工作流规范

  • 双模态协议：学习模式（不限 Token）vs 应用模式（极限节流）
  • 开源学习流程：GitHub 搜 Repo → 读代码 → 教用户
  • 保存 GitHub Token，配置 gh CLI

18:24 - 深度学习 OpenClaw

通过官方文档学习了 16 个核心模块：架构、安装、配置、渠道系统、模型提供商、Agent Runtime、会话管理、心跳机制、Cron 定时任务、Webhooks、Hooks、多智能体路由、安全模型、插件系统、记忆系统、远程访问。

成果： skills/openclaw-mastery/SKILL.md（9KB）

18:56 - 用强模型为本地模型创建技能

用 mimo-v2-pro 创建了 5 个基础技能供本地 qwen3.5:9b 复用：

  1. 代码助手
  2. 文本处理
  3. 数据分析
  4. 文件管理
  5. 网络研究

19:12 - 搭建博客

框架选型（Hexo + Butterfly）→ 项目初始化 → GitHub Actions CI/CD → 部署上线。

成果： https://diodeshing.github.io/my-blog/

19:56 - 内容生产

一口气写了 10 篇技术博客，涵盖 Python 技巧、终端美化、Markdown 速成、Git 规范、编程语言推荐、Docker 入门、GitHub Actions、API 设计、AI 写代码、正则表达式。

20:15 - 博客更名

从「极客代码花园」改为「击穿二极管」。

20:56 - 深度网页抓取

学习 Jina Reader API + Python httpx + BeautifulSoup，实现网页内容提取。

21:04 - 第二大脑联动

学习 Notion API + Obsidian 集成，安装 notion-client。

21:09 - 数据库分析

学习 SQLite/MySQL/PostgreSQL 直连，安装 sqlite-utils。

21:16 - 自动化通知

学习 Email (SMTP) + Telegram Bot + Webhook 推送，配置 QQ 邮箱并测试成功。

21:31 - 写每日总结

你正在读的这篇文章。

🛠 今日创建的技能

💡 今日收获

  1. AI 不是替代者，是加速器 — 它负责执行，人类负责决策
  2. 技能沉淀是关键 — 第一次做要花时间，第二次就是秒级
  3. 强模型教弱模型 — 用大模型学技能，存成本地文件，小模型直接用
  4. 自动化是终极效率 — Cron + 推送 + 博客 = 无人值守内容流水线

📅 明天计划

  • 配置 Telegram Bot 推送
  • 设置 Cron 每日新闻推送
  • 整理 Obsidian 笔记 → 博客文章
  • 探索更多 ClawHub 技能
