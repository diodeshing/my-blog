AI 智能体帮我配好了整个开发环境

今天做了一个实验：完全不自己动手，让 AI 智能体帮我搭建开发环境。

结果是：一个小时之内，我从一台全新的电脑，变成了一个拥有完整开发环境、GitHub 工具链、飞书集成、甚至还有一个上线博客的开发者。

传统配环境有多痛苦

每个开发者都经历过这样的场景：

  1. 装 Node.js，版本不对，卸了重装
  2. 配 Git，SSH key 生成了但不知道放哪
  3. 装 VS Code 插件，装了一半忘了装了哪些
  4. 配置 SSH、代理、环境变量，搞了一下午
  5. 最后发现某个工具漏装了，又从头排查

整个过程充满了重复劳动和上下文切换。而这恰好是 AI 最擅长的事。

AI 智能体做了什么

这次我用的是 OpenClaw（一个 AI 智能体框架），让它接管了所有配置工作。

1. 安装工具链

不需要我查命令，不需要我记参数，AI 直接搞定。

2. 搭建博客

从零到上线，全自动。甚至还帮我写了第一篇博文。

3. 学习与沉淀

以后再遇到相关问题，AI 可以直接读取这些知识，不用重新搜索。

核心原理：技能沉淀

AI 智能体有个关键设计：每次完成任务后，把操作步骤保存为”技能”。

这意味着：

  • 第一次做：AI 需要搜索、尝试、调试
  • 第二次做：AI 直接读技能文件，秒级完成
  • 知识不会丢失：换一个 AI 模型也能用同样的技能

实际效果

一个小时的配置成果：

工具/服务 状态
GitHub CLI + 认证 ✅ 已就绪
飞书集成 ✅ 已连接
Hexo 博客 ✅ 已部署
GitHub Actions CI/CD ✅ 自动运行
10+ 个自定义技能 ✅ 已保存

一些思考

AI 擅长的事

  • 执行重复性操作（安装、配置、部署）
  • 记忆和组织大量信息
  • 按流程一步步执行复杂任务

AI 不擅长的事

  • 做审美判断（这个配色好不好看）
  • 理解模糊需求（”帮我弄好看点”）
  • 创意性工作（需要你给方向）

最佳协作模式

总结

最好的工具不是替代你，而是让你把时间花在真正需要创造力的地方。

AI 智能体帮我做了所有我不想做的琐事。它不累，不会忘记步骤，而且每次学习后都会变得更强。

如果你也想试试，可以从 OpenClaw 开始。
