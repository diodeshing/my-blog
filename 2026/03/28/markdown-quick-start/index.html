常见坑

  1. 换行：行尾加两个空格，或者空一行
  2. 缩进：代码块内缩进用空格，不要用 Tab
  3. 特殊字符：* # > 等有特殊含义，需要时加 \ 转义

总结

Markdown 的精髓是”内容优先，格式其次”。你不需要记住所有语法，常用的就那 10 个，写多了自然就记住了。
